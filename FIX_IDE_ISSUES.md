# Решение проблемы с пустыми файлами DAO в IntelliJ IDEA

## Проблема
IntelliJ IDEA показывает файлы DAO интерфейсов как пустые, хотя они содержат код.

## Решение

### Шаг 1: Очистка кэша IDE
1. В IntelliJ IDEA: **File** → **Invalidate Caches / Restart**
2. Выберите **Invalidate and Restart**
3. Дождитесь перезапуска IDE

### Шаг 2: Переиндексация проекта
1. **File** → **Project Structure** (Ctrl+Alt+Shift+S)
2. Убедитесь, что модуль правильно настроен
3. **File** → **Settings** → **Build, Execution, Deployment** → **Compiler** → **Java Compiler**
4. Проверьте версию Java (должна быть 23)

### Шаг 3: Пересборка проекта
1. **Build** → **Rebuild Project**
2. Дождитесь завершения пересборки

### Шаг 4: Проверка файлов
Убедитесь, что все файлы DAO существуют в папке:
```
src/main/java/com/example/insurancecompany/dao/
```

Должны быть следующие файлы:
- `OwnerDao.java`
- `VehicleDao.java`
- `TariffDao.java`
- `PolicyDao.java`
- `InsuranceDao.java`
- `PayoutDao.java`
- `StatusDao.java`
- `UserAccountDao.java`

### Шаг 5: Если проблема сохраняется
1. Закройте проект
2. Удалите папку `.idea` в корне проекта
3. Откройте проект заново
4. IntelliJ IDEA пересоздаст конфигурацию

### Альтернативное решение
Если файлы действительно пустые в файловой системе:
1. Откройте файл в любом текстовом редакторе (Notepad++, VS Code)
2. Убедитесь, что файл содержит код
3. Если файл пустой, скопируйте код из этого проекта

## Проверка
После выполнения всех шагов:
1. Откройте любой файл DAO (например, `OwnerDao.java`)
2. Убедитесь, что видите код интерфейса
3. Проверьте, что нет ошибок компиляции
